Cinema: Express Basics

  • October 30, 2022

After getting the server and static site up and running, I felt comfortable switching to a data set with more movie data.

🔗 Cinema on Github

Goals

  • Switch to a data set with more information that can be displayed on our results page.
  • Pre-load movie data on the search page, so that checkboxes can be dynamically assigned with keywords present in the data set.

Express Basics

As issues started arise for the front-end team when they needed to access the style sheet on the index page, I realized that I needed to go back and learn some Express fundamentals.

Defining Routes and Sending Data:

  • For Cinema, I will be using a GET request to serve the static home page and a POST request to send the search criteria to the results page.

  • On a GET request to the home page, I use a callback to parse the local csv data, send that data to a javascript function to make modifications, and render the home page with search terms so that genre checkboxes can be created dynamically.

      app.get('/', (req, res) => {
  var parsedData = [];
  //Using csv-parser to parse csv data to array of objects
  fs.createReadStream(csvFilePath)
    .pipe(csvParser())
    .on('data', (data) => {
      parsedData.push(data);
    })
    .on("end", () => {
      //Extract genres from movie data
      let genres = modify.genreData(parsedData);
      //Render index page, send genres to the page
      res.render('index', { genres }) 
    })
  })

  • On a POST request to the results page, I had to parse the csv data again because I couldn’t find a way to save the already parsed and modified data from the previous route. I know that there is likely a better way to do this, but because our project was already at the midway point, I wanted to keep moving forward.

  • After parsing the data, I pass it to a javascript function that searches for matching movies. The results page is then rendered with result data

File Structure and Serving Static Files:

  • After the front end team ran into some issues access stylesheets, I cleaned up the file structure to align with Express Documentation:

        >movieData
      - movies.csv
    >public/css
      - styles.css
    >services
      - modify.js
      - search.js
    >uploads
    >views
      - index.ejs
      - results.ejs
    - server.js

  • Express Documentation also helped with the correct way to access static files by defining a public directory:

  app.use(express.static(path.join(__dirname, 'public')));

After creating a route to the public directory, stylesheets are accessed like this:

  <link rel="stylesheet" type = "text/css" href="css/styles.css">

        //Return all genres present in the dataset
  function genreData(movies){
      let modifiedMovies = movieData(movies);
      genreArr = [];
      modifiedMovies.forEach(movie => {
          movie.genres.forEach(genre => {
              if(!genreArr.includes(genre)) genreArr.push(genre);
          });
      });
      return genreArr;
  }
