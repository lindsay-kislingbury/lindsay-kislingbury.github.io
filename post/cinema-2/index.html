Cinema: So Many AHA! Moments

  • 30/10/2022
  • 6-minute read

The team decided to implement a larger data set that includes more information that they can display on the results page.

Goals

  • Switch to a data set with more information that can be displayed on our results page.
  • Pre-load movie data on the search page, so that checkboxes can be dynamically assigned with keywords present in the data set.
  • Get search terms from the index page, send those search terms along with the matching movies to the results page.

Express Basics

As issues started arise for the front-end team when they needed to access the style sheet on the index page, I realized that I needed to go back and learn some Express fundamentals.

Defining Routes and Sending Data:

  • For Cinema, I will be using a GET request to serve the static home page and a POST request to send the search criteria to the results page.

  • On a GET request to the home page, I use a callback to parse the local csv data, send that data to a javascript function to make modifications, and render the home page with search terms so that genre checkboxes can be created dynamically.

        //Return all genres present in the dataset
  function genreData(movies){
      let modifiedMovies = movieData(movies);
      genreArr = [];
      modifiedMovies.forEach(movie => {
          movie.genres.forEach(genre => {
              if(!genreArr.includes(genre)) genreArr.push(genre);
          });
      });
      return genreArr;
  }

Searching For Movies

  • First, I created a modified array of movie data that includes only the data I need for searching:
    let movieSearchList = [];
    movieData.forEach(movie => {
        movieSearchList.push({
            movieId: movie.id,
            keywords: movie.keywords,
            genres: movie.genres
        })
    });

  • Because the list of keywords present in the entire dataset was fairly large, I wouldn’t be able to display all available keywords on the search page to choose from.
    The solution we landed on was to allow the user to type in a keyword or two, and do a fuzzy search on our existing list of keywords in the data set.

  • A fuzzy search is an approximate search, we didn’t want to match keywords exactly. If a user searched “pirates”, we also wanted to return movies that had keywords like “pirate ship” and “pirate”

  • I ended up using a library called Fuse.js. The library is lightweight, has no dependencies, and allows you to set a few options like case sensitivity and the ability to sort by match score.

      const Fuse = require('fuse.js'); 
  var options = { //Fuse options
      includeScore: false,
      isCaseSensitive: false,
      includeMatches: false,
      shouldSort: true,
      findAllMatches: true,
      threshold: 0.3,
      keys: ['keywords'],
  }

How the search works

1. Use Fuse.js to find all movie id’s for movies matching the keywords input by the user

  //Keyword Search. Store matching movieIds in array
  let fuse = new Fuse(movieSearchList, options);
  let keywordMatches = [];
  var keywordMatchFound = false;
  if(searchKeywords){//Find keyword matches using fuse.js
      if(searchKeywords){
          let result = fuse.search(searchKeywords);
          if(result){
              result.forEach(result => {
                  keywordMatches.push(result.item.movieId);
              })
          }
      }
      if(keywordMatches.length > 0){
          keywordMatchFound = true;
      } else {
          keywordMatchFound = false;
      }
  }

2. Find all movie id’s for movies matching genres input by the user. This is an exact match to the genre name associate with a checkbox on the search page
