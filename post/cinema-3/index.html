Cinema: Completing a Project

  • November 12, 2022

I decided on a fuzzy search of the database for matching movies and needed to send the results to the front end in a way that was easy to access.

I also needed to create a simple way to display such a large amount of data.

🔗 Cinema on Github

Goals

  • Get search terms from the index page, send those search terms along with the matching movies to the results page.
  • Style the two html pages so that they are intuitive, clear, and functional.
  • Complete the project, even if it isn’t 100% perfect.

Searching For Movies

  • First, I created a modified array of movie data that includes only the data I need for searching:
    let movieSearchList = [];
    movieData.forEach(movie => {
        movieSearchList.push({
            movieId: movie.id,
            keywords: movie.keywords,
            genres: movie.genres
        })
    });

  • Because the list of keywords present in the entire dataset was fairly large, I wouldn’t be able to display all available keywords on the search page to choose from.
    The solution we landed on was to allow the user to type in a keyword or two, and do a fuzzy search on our existing list of keywords in the data set.

  • A fuzzy search is an approximate search, we didn’t want to match keywords exactly. If a user searched “pirates”, we also wanted to return movies that had keywords like “pirate ship” and “pirate”

  • I ended up using a library called Fuse.js. The library is lightweight, has no dependencies, and allows you to set a few options like case sensitivity and the ability to sort by match score.

      const Fuse = require('fuse.js'); 
  var options = { //Fuse options
      includeScore: false,
      isCaseSensitive: false,
      includeMatches: false,
      shouldSort: true,
      findAllMatches: true,
      threshold: 0.3,
      keys: ['keywords'],
  }

How the search works

1. Use Fuse.js to find all movie id’s for movies matching the keywords input by the user

  //Keyword Search. Store matching movieIds in array
  let fuse = new Fuse(movieSearchList, options);
  let keywordMatches = [];
  var keywordMatchFound = false;
  if(searchKeywords){//Find keyword matches using fuse.js
      if(searchKeywords){
          let result = fuse.search(searchKeywords);
          if(result){
              result.forEach(result => {
                  keywordMatches.push(result.item.movieId);
              })
          }
      }
      if(keywordMatches.length > 0){
          keywordMatchFound = true;
      } else {
          keywordMatchFound = false;
      }
  }

2. Find all movie id’s for movies matching genres input by the user. This is an exact match to the genre name associate with a checkbox on the search page

HTML & CSS

  • This will be my first attempt at styling a web page. Initally, I was tempted to use bootstrap, but ultimately went with html/css only. I want to build a good foundation in the basics.

  • I cannot overstate how helpful this guide A Complete Guide to Flexbox was for learning how to use flexboxes. I relied almost entirely on this guide.

  • I started by creating a header container, main container for the body, and two containers for the sections:

  • Getting the checkboxes to align correctly inside the right-side flex box became a nightmare, and at one point I had created so many nested flexboxes that I had difficulty creating unique names. Eventually I scaled everything back to the bare minimum and ultimately used just one outer container for the section, and an inner container for the checkboxes.

Expanding Divs

  • I struggled to find the best way to display so much data on the results page effectively. If a user returned a search with more than 4 movies, the list of results became very large.

  • I played around with the idea of expanding divs, but wanted to avoid adding any client-side javascript that would further complicate the project. Luckily, it turns out that expanding divs are a native html element!

  • Here is a look at how the data sent from the server to the results page is accessed with EJS:

<% if(searchResults.matchFound) { %>
            <ul class = "flex-results-container">
            <% for(var i=0; ((i < 31) && (i < searchResults.bestMatches.length)); i++) { %>
                <li class = flex-result>
                    <details>
                        <p>
                        <b>Release Date: </b><%-searchResults.bestMatches[i].releaseDate; %> <br>
                        <b>Original Language: </b><%-searchResults.bestMatches[i].language; %><br>
                        <b>Run Time: </b><%-searchResults.bestMatches[i].runtime; %> minutes <br>
                        <b>Overview: </b><%-searchResults.bestMatches[i].overview; %> <br>
                        <b>Genres: </b><% for(var j=0; j < searchResults.bestMatches[i].genres.length; j++) { %>
                            <%- searchResults.bestMatches[i].genres[j] %>
                        <% } %> <br>
                        <b>Tags: </b> <% for(var j=0; j < searchResults.bestMatches[i].keywords.length; j++) { %>
                            <%- searchResults.bestMatches[i].keywords[j] %>
                        <% } %> 
                        </p>
                    <summary>
                        <h3> <%-searchResults.bestMatches[i].title; %> </h3>

                    </summary>   
                    </details>
                </li>
            <% } %>
            <% } %>
            </ul>
