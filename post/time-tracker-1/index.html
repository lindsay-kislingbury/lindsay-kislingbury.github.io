Time Tracker: Sessions and Authentication

  • March 26, 2023

I am starting on a new project with a focus on maintaining user sessions. The site will allow the user to log in and track time. The previous time stamps will be saved in a database and displayed on a dashboard when the user logs in.

I have already made significant progress on the project, I currently have a functioning site that stores the user’s session, sends a timestamp to the database, and pulls that data to display on a user’s dashboard.

What I’m Using

  • Node/Express
    I am using Node with Express again in this project. I want to explore further and build a full stack site with a more defined client/server structure.

  • MongoDB
    I chose MongoDB for this project because I don’t forsee the need for a complicated database structure. MongoDB’s flexibility allows me to store and update an array of timestamps within a user document and it will allow me to add other data I may want to store for each user without knowing the future of the project.

  • Mongoose
    Mongoose does some of the heavy lifting when working with MongoDB in Node. It includes query building and type casting that will make working with MongoDB for the first time a bit smoother.

  • PassportJS
    Middleware for Express with built in strategies for various types of authentication. For now, I will be using a local strategy, that accesses user credentials in the database.

  • bcrypt
    A node library that I am using to hash user passwords before they are stored in the database.

  • express-session
    Another node library. express-session stores the sessionID in the HTTP cookie, and the session data is stored in the database.

How it’s going

My initial focus was to get the user sessions and database working.

The passport local strategy looks to my mongo database for user credentials. Mongoose schemas allow for methods to be associated with a schema. I created a method that compares the password entered to the hashed password stored in the database.

  passport.use(
    new LocalStrategy(
      {
          usernameField: 'username',
          passwordField: 'password'
      },
      async (username, password, done) => {
        try {
          const user = await User.findOne({"username": username});
          if (!user) return done(null, false);

          const passwordMatch = await user.comparePassword(password);
          if (!passwordMatch) return done(null, false);

          return done(null, user);
        } 
        catch (err) {
          return done(err);
        }
      }
    )
  );

  UserSchema.methods.comparePassword = async function(password) {
      try{
          let result = await bcrypt.compare(password, this.password);
          return result;
      }
      catch (error) {
          console.error(error);
          return false;
      }
  };

Once I had authentication working, I needed to create sessions to keep a user logged in. As I already had express-session configured, I was able to use the passport session feature. Passport session populates the request body with the current user.
