Time Tracker: CRUD

  • July 2, 2023

When I started this project, I chose Mongodb simply because I had seen it used frequently in other projects similar to mine. I was already familiar with mySQL, but I had never worked with a non-relational db before.

Although it did cause some serious headaches, It was the right choice for this project. It’s flexibility allowed me to change the structure of the data as the project evolved, and ultimately, I think it eased the pain of my learning curve.

Before I describe the CRUD operations, Here’s a look at the User schema

//User.js
const UserSchema = new Schema(
    {
        username: String,
        name: String,
        password: String,
        timestamps: [{
            title: String,
            project: String,
            tags: [String],
            date: Date,
            elapsedTime: Number,
        }],
        createdAt: { 
            type: Date, 
            default: Date.now 
        },
        expireAt: { 
            type: Date, 
            default: undefined 
        } 
    },
    {
        collection: 'users'
    }
);

I am using Mongoose ODM to simplify a lot of the operations. Mongoose allows me to enforce a schema for the data, and comes with some validation and other features that made my life a bit easier.

You’ll notice that the timestamps array holds an array of all entries. With a traditional db like mySQL, I would have made a separate table of entries, with keys to associate them to their users.

Figuring out how to access nested entries in this array was frustrating. However, once I got a handle on it, it was really convienent to treat the whole schema as an object containing everything I need for each user.

Passwords are encrypted before they are stored. I’ll discuss that in another entry.

Creating A New Entry

On the Front End

A user can create a new entry two ways: with the timer, or by manual entry.

When a user clicks the save button on the stopwatch or manual entry widget, the send function is called with the type of entry send as a parameter.

The appropriate build function is called to create the post request. These functions simply grab the values from input fields on the widget and create an object that will be sent in POST.

//content-helpers.js
async function send(type){
    if(!$('#'+type+'-project').val()){
        $('#'+type+'-project').addClass('is-invalid')
    }
    var valid = validateInputs(type);
    var postData;
    if(valid){
        if(type == 'new'){
            postData = buildNewEntry();
        }else if(type == 'add'){
            postData = buildAddEntry();
        }else{
            console.log('invalid submit type');
        }
        var post = JSON.stringify(postData);
        var xhr = new XMLHttpRequest();
        xhr.open('POST', '/time/create', true);
        xhr.setRequestHeader('content-type', 'application/json;charset=UTF-8');
        xhr.send(post);
        clearAllInputs();
        update();
    }
}

On the Back End

This is where Mongodb with Mongoose really shine. I simply push the new object into the array of users.

//TimeController.js
const createStamp = async (req, res) => {
   try{
      const {title, project, tags, date, elapsedTime} = req.body;
      const timestamp = {
         title: title,
         project: project,
         tags: tags,
         date: date,
         elapsedTime: elapsedTime,
      };
      const user = await User.findById(req.user.id);
      user.timestamps.push(timestamp);
      await user.save();
      res.status(200).json({message: "Stamp created successfully"});
   } catch (error) {
      console.log(error);
      res.status(500).json({message: "Error creating stamp"})
   }
};

Editing An Entry

Here’s where most of the learning happened. While it’s great that Mongodb allows me to simply store a whole array inside of a User model, as opposed to linking two tables with keys…. I spent a lot of time in the MongoDB and Mongoose docs figuring out how to edit a specific element in that nested array.

Here’s how I solved the problem:

In MongoDB, the $set operator replaces the value of a field with a specified value.
$ identifies an element in an array to update without explicitly specifying the position.

In the findOneAndUpdate method, I locate the user by id, and then the specific entry by id.
I use $set to update the values of all fields in the entry.

//TimeController.js
const editStamp = async(req,res) => {
   try{
      const editId = mongoose.Types.ObjectId(req.body.stampId);
      const timestamp = {
         title: req.body.title,
         project: req.body.project,
         tags: req.body.tags,
         date: req.body.date,
         elapsedTime: req.body.time,
      }
      await User.findOneAndUpdate({_id: req.user.id, 'timestamps._id': editId},
         {$set: {'timestamps.$': timestamp}}
      );
      res.status(200).json({message: "Stamp edited successfully"});
   } catch(error) {
      console.log(error);
      res.status(500).json({message: "Error editing stamp"})
   }
}

Deleting An Entry

Deleting an entry is very similar to the process used for editing.

function remove(){
    var stampId = document.getElementById('button-remove').value;
    $.ajax({
        type: 'POST',
        url: '/time/remove',
        cache: false,
        data: {
            deleteId: stampId,
        }
    })  
    clearAllInputs();
    update();
    $('#editModal').modal('hide');
}

Instead of $set, the $pull operator is used to remove the entire entry from the array.

const removeStamp = async(req, res) => {
   try{
      const deleteId = mongoose.Types.ObjectId(req.body.deleteId);
      await User.updateOne({ _id: req.user.id }, {
         $pull: {timestamps: {_id: deleteId}}
      });
      res.status(200).json({message: "Stamp removed successfully"});
   } catch(error){
      console.log(error);
      res.status(500).json({message: "Error removing stamp"})
   }
}

Updating The View

Every time a user creates, edits, or deletes a new entry, another GET request is made for the current db data. All fields and the list of recent entries is reloaded.
